- [x] 1. 完成共享规则引擎与核心杭州麻将规则
  - 已完成共享规则引擎拆分、状态机统一、番型结算、财飘链、副露来源记录，对应设计文档 Shared Rule Engine 与 Requirement 2、Requirement 5、Requirement 8。
  - 已完成规则回归测试、AI 基础行为测试和日志事件测试，为后续正式上线迭代提供稳定基线。

- [x] 2. 完成单机、基础联机与工程化基线
  - 已完成移动端基础牌桌、房间创建加入、实时同步、重连基础流程、结构化事件、工程化拆分与部署检查脚本。
  - 已完成联机鉴权收口、AI 补位收口、静态资源 404 策略和部署冒烟检查增强。

- [ ] 3. 建立正式上线级配置、观测与运行安全体系
  - 补齐环境变量校验、版本信息输出、服务健康检查、请求日志、错误日志和房间生命周期清理机制，对应设计文档 Game Server、Error Handling。
  - 增加基础限流、请求体保护、超时保护和 WebSocket 连接上限，降低公开部署后的滥用风险。
  - [x] 3.1 实现服务健康检查与版本信息接口
    - 新增如 `/api/health`、`/api/version` 的只读接口。
    - 输出运行状态、版本号、启动时间和房间统计摘要。
  - [x] 3.2 实现结构化日志与错误采集
    - 为 HTTP、WebSocket、房间创建、开局、动作拒绝、重连失败增加统一日志事件。
    - 为服务端异常补齐 request id、room code、player id 维度信息。
  - [x] 3.3 实现房间清理与资源回收
    - 增加空房间回收、结算后房间保留时长和长期未活动房间清理。
    - 避免 `rooms` 与 `roomSockets` 长期增长。
  - [x] 3.4 实现基础限流与输入保护
    - 对建房、加房、动作、重连和 WebSocket 升级增加频率限制。
    - 限制昵称长度、请求体大小和异常参数格式。
  - [x]* 3.5 为服务稳定性能力补充测试
    - 覆盖健康检查、版本接口、空房回收和限流拒绝行为。

- [ ] 4. 强化联机可靠性与会话恢复能力
  - 从正式上线目标出发，完善断线恢复、页面刷新恢复、重复连接替换和房间内在线状态一致性，对应设计文档 Realtime Room Adapter、Error Handling。
  - 让联机客户端在弱网、刷新页面和服务短暂重启后保持更稳定的恢复路径。
  - [x] 4.1 持久化房间会话与页面刷新恢复
    - 将 `roomCode`、`playerId`、`sessionToken` 和必要房间态持久化到本地存储。
    - 页面初始化时尝试自动恢复到房间页或牌桌页。
  - [x] 4.2 实现重复连接替换与在线状态仲裁
    - 同一玩家新连接建立后替换旧连接，避免重复广播与在线状态错乱。
    - 明确一个玩家的唯一活跃 WebSocket 语义。
  - [x] 4.3 增强重连退避和恢复提示
    - 为重连增加退避上限、恢复成功提示和长期失败后的明确操作引导。
    - 保证轮询与 WebSocket 之间切换不会造成多重定时器残留。
  - [x] 4.4 补齐联机恢复与一致性集成测试
    - 覆盖刷新恢复、重复连接替换、服务端短时中断后的恢复路径。

- [ ] 5. 打磨正式上线级移动端交互与可用性
  - 继续提升手机端操作效率、误触防护、可读性、结算说明和弱网提示，对应设计文档 Web App UI、Client Game Controller。
  - 目标是让普通用户无需解释即可完成单机和联机完整对局。
  - [x] 5.1 完善房间页与牌桌页的信息层级
    - 增强房主、庄家、当前操作者、在线状态、等待状态和累计战绩的可视化。
    - 优化弱网、重连中、动作失败等状态提示文案。
  - [x] 5.2 增加正式版结算详情与回放入口
    - 在结算页展示番型拆解、输赢来源、庄家倍率说明和财神/财飘明细。
    - 从结算页进入最近一局事件回放摘要。
  - [x] 5.3 完善移动端操作体验
    - 优化按钮热区、长列表滚动、横竖屏布局、低分辨率牌面可读性和操作反馈节奏。

  - [x]* 5.4 为正式版 UI 行为补充测试
