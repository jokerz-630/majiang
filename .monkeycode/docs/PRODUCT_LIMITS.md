# 杭州麻将 Web 游戏产品限制与后续演进说明

## 1. 当前产品定位

当前版本是一个移动端优先的杭州麻将 Web 游戏，目标是提供：

1. 手机浏览器直接可玩的单机模式
2. 基于房间号和游客昵称的好友联机模式
3. 围绕当前规则实现的正式可发布质量门禁

当前版本更适合小范围好友开房体验和规则验证型上线，不包含账号体系、支付体系、排行榜体系和多实例房间基础设施。

## 2. 当前规则覆盖范围

### 2.1 已支持牌组与基础流程

当前规则引擎已覆盖：

1. 杭州麻将基础牌组
   - 万、条、筒
   - 东、南、西、北
   - 中、发、白
2. 正常发牌、摸牌、出牌
3. 吃、碰、杠、过
4. 自摸胡
5. 流局结算

### 2.2 已支持的杭州麻将核心约束

当前版本已实现以下业务约束：

1. 白板是财神，按百搭牌参与胡牌判定
2. 杭州麻将当前版本无点炮，胡牌只通过自摸成立
3. 吃牌次数上限为 2 次
4. 碰牌次数不限
5. 庄家参与输赢按 8 倍结算
6. 牌桌按东南西北四个座位组织
7. 新摸牌通过 `drawnTileId` 做视觉区分
8. 打出白板后进入财飘等待，下一次自己摸牌能胡才累计财飘

### 2.3 已支持的番型与翻数口径

当前代码与测试已覆盖：

1. 平胡 1 翻
2. 有财神的敲响 2 翻
3. 七小对子无财神 4 翻
4. 七小对子有财神 2 翻
5. 七小对子有财神且敲响 4 翻
6. 三财胡额外 1 翻
7. 四财胡额外 2 翻
8. 财飘 1 次额外 1 翻
9. 财飘 2 次额外 2 翻

## 3. 当前联机能力范围

### 3.1 已支持能力

当前联机版本已支持：

1. 创建房间
2. 输入六位房间号加入房间
3. 房间等待页准备状态同步
4. 房主开局
5. 服务端权威状态校验
6. WebSocket 广播玩家视角牌局快照
7. 页面刷新后会话恢复
8. 断线重连与重复连接替换
9. 房间累计战绩展示

### 3.2 当前联机身份模型

当前联机首版使用轻量会话模型：

1. 玩家通过游客昵称进入房间
2. 服务端生成 `playerId`
3. 服务端生成 `sessionToken`
4. 前端在本地存储 `roomCode`、`playerId`、`sessionToken`

这个模型适合轻量联机体验，适合当前小规模上线目标。

## 4. 当前产品限制

### 4.1 规则限制

当前版本未覆盖以下本地变体或扩展规则：

1. 点炮胡
2. 承包
3. 飘财扩展规则
4. 爆头
5. 杠开
6. 抢杠胡
7. 其他未进入当前测试集的地方性细则

这些扩展项在设计文档中已经被标记为后续通过规则配置扩展。

### 4.2 人数与桌型限制

当前版本固定按四人麻将设计：

1. 联机桌型固定四座位
2. 房间真人不足时自动由 AI 补位
3. 当前没有二人局和三人局分支
4. `fillWithAi` 当前在服务端被固定为 `true`

### 4.3 房间与身份限制

当前游客房间存在以下限制：

1. 没有账号登录
2. 没有跨设备身份绑定
3. 没有正式好友系统
4. 没有永久房间记录
5. 浏览器本地会话清理后需要重新加入房间
6. 依赖本地保存的 `sessionToken` 恢复会话

### 4.4 架构与部署限制

当前部署能力存在以下边界：

1. 房间状态保存在内存中
2. 进程重启后房间会丢失
3. 当前适合单实例部署
4. 当前没有跨实例共享房间能力
5. 当前没有数据库持久化
6. 当前没有外部缓存或消息总线

### 4.5 运维限制

当前版本的运行观测已经具备基础能力，但仍存在上限：

1. 健康检查只返回进程级摘要
2. 当前没有 Prometheus 指标
3. 当前没有分布式追踪
4. 当前没有独立告警系统
5. 当前日志默认写到标准输出

## 5. 当前适用场景

当前版本适合以下场景：

1. 小规模正式体验
2. 规则验证型上线
3. 好友邀请开房体验
4. 单机练习与 AI 对局
5. 后续继续演进的稳定基线

## 6. 后续演进建议

### 6.1 规则引擎方向

优先级较高的演进项：

1. 把更多杭州麻将地方细则抽象成可配置规则项
2. 为额外番型补充独立判定单元与测试
3. 为承包、爆头、杠开、抢杠胡等规则建立扩展开关
4. 保持规则引擎与单机、联机、回放共用同一套状态机

### 6.2 联机产品方向

建议的下一阶段演进：

1. 增加账号体系或轻量绑定身份机制
2. 增加好友关系、历史房间记录和战绩页
3. 支持房间列表、观战或最近房间恢复
4. 增强断线超时、托管与默认动作机制

### 6.3 服务端架构方向

如果目标升级到更大规模正式运营，建议按以下顺序扩展：

1. 为房间和会话引入持久化存储
2. 为多实例部署引入共享状态层
3. 把房间清理、限流和会话恢复策略外置配置化
4. 引入更完整的监控、告警和指标体系

### 6.4 前端体验方向

下一阶段可以继续增强：

1. 更完整的结算回放
2. 更明确的等待计时与托管提示
3. 横屏与超小屏的进一步优化
4. 新手引导与规则说明页
5. 更丰富的动作反馈和动画节奏

### 6.5 AI 与质量保障方向

继续演进建议：

1. 增强 AI 对向听、搭子和收益的评估能力
2. 增加更多长局模拟样本数和失败样本归档
3. 增加到结算页的完整浏览器端到端路径
4. 在发布门禁中逐步加入更多回归场景

## 7. 对外说明建议

当前版本对外发布时，建议明确传达以下信息：

1. 当前为杭州麻将首版正式可用 Web 版本
2. 当前主打单机与好友房体验
3. 当前房间采用游客会话机制
4. 当前规则以已实现和已测试范围为准
5. 更多地方性规则与扩展玩法会在后续版本持续补齐
